# Telegram Bot для системы заказа масел

Этот модуль добавляет функционал Telegram-бота к существующему веб-приложению для заказа масел.

## Возможности бота

- 📖 **Просмотр каталога** - отображение всех доступных масел с ценами
- 🛒 **Оформление заказа** - пошаговый процесс заказа с выбором товаров
- 📋 **Мои заказы** - просмотр истории заказов
- 📞 **Контакты** - контактная информация компании
- 🔧 **Админ-панель** - управление заказами (для авторизованных пользователей)

## Установка и запуск

### 1. Получение токена бота

1. Откройте Telegram и найдите бота [@BotFather](https://t.me/BotFather)
2. Отправьте команду `/newbot`
3. Введите имя бота (например, `OilPress Bot`)
4. Введите username бота (должен заканчиваться на `bot`, например, `oilpress_shop_bot`)
5. Скопируйте полученный токен (выглядит как `1234567890:ABCdefGHIjklMNOpqrsTUVwxyz`)

### 2. Настройка переменных окружения

Создайте файл `.env` в корне проекта на основе `.env.example`:

```bash
cp .env.example .env
```

Отредактируйте файл `.env` и укажите ваш токен:

```env
TELEGRAM_BOT_TOKEN=1234567890:ABCdefGHIjklMNOpqrsTUVwxyz
```

### 3. Запуск через Docker Compose

Запустите все сервисы (база данных, веб-приложение и бот):

```bash
docker compose up --build
```

Или запустите только бота:

```bash
docker compose up bot
```

### 4. Локальный запуск (без Docker)

Установите зависимости:

```bash
pip install -r requirements.txt
```

Запустите бота:

```bash
python bot.py
```

## Использование

### Команды бота

- `/start` - Запустить бота и показать главное меню
- `/catalog` - Показать каталог масел
- `/order` - Начать оформление заказа
- `/myorders` - Просмотреть мои заказы
- `/contacts` - Контактная информация
- `/help` - Справка по командам
- `/admin` - Панель администратора

### Процесс оформления заказа

1. Нажмите кнопку "🛒 Оформить заказ" или отправьте команду `/order`
2. Введите имя
3. Введите фамилию
4. Введите номер телефона
5. Введите адрес доставки
6. Выберите товары из каталога (нажимая на кнопки)
7. Нажмите "✅ Завершить выбор"
8. Введите дату доставки в формате ГГГГ-ММ-ДД
9. Выберите статус заказа
10. Подтвердите заказ

## Архитектура

Бот использует те же модели данных и функции базы данных, что и веб-приложение:
- `Catalog.py` - работа с каталогом товаров
- `Database.py` - операции с базой данных
- `Customer.py`, `OilOrder.py`, `OrderItem.py` - модели данных

Это обеспечивает согласованность данных между веб-интерфейсом и Telegram-ботом.

## Структура файлов

```
Oil/

```

## Требования

- Python 3.8+
- aiogram 3.x
- python-dotenv
- MySQL (уже используется в проекте)

## Примечания

- Бот работает асинхронно и использует машину состояний (FSM) для процесса оформления заказа
- Все данные сохраняются в общую базу данных с веб-приложением
- Для продакшена рекомендуется настроить логирование и обработку ошибок
